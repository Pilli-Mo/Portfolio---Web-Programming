Einführung

In der dritten Vorlesung haben wir uns erneut kurz HTML gewidmet – diesmal mit Fokus auf das HT (Hypertext) in HTML. Dabei wurde der <a>-Tag behandelt, mit dem Hyperlinks zu anderen Seiten oder Dokumenten erstellt werden können.

Der <a>-Tag im Detail

Der <a>-Tag, auch Anker-Tag genannt, wird verwendet, um Links zu erstellen. Diese können entweder auf externe Websites oder auf interne Seiten derselben Website verweisen.

Grundlegender Aufbau

<a href="https://link.de">Linktext</a>

Attribute des <a>-Tags

  • href: Ziel-URL des Links (Pflichtattribut).
  • target: Legt fest, ob der Link in einem neuen Tab (_blank) oder im gleichen Fenster (_self) geöffnet wird.
  • title: Zusätzliche Information, die beim Überfahren des Links erscheint.

Links zu internen Seiten

Neben externen Websites können Links auch auf andere HTML-Dateien innerhalb derselben Website verweisen. Das ist besonders nützlich, wenn man eine mehrseitige Website mit Unterseiten erstellt. 

<a href="elixiere.html">Elixiere</a>

Empfohlene Ordnerstruktur

Um eine übersichtliche Navigation zu gewährleisten, ist eine strukturierte Ablage wichtig. Eine mögliche Struktur: 

index.html            (Startseite)
bar/bar.html          (Unterseite "Bar")
getraenke/getraenke.html  (Unterseite "Getränke")
images/               (Ordner für Bilder)

Verlinken zwischen verschiedenen Ordnern

Wenn eine Seite aus einem Unterordner auf eine Datei in einem anderen Ordner verweisen soll, muss der vollständige relative Pfad angegeben werden: 

<a href="getraenke/getraenke.html">Getränke</a>

Verlinkung auf übergeordnete Seite

Mit dem Präfix ../ gelangt man eine Ebene im Verzeichnisbaum nach oben: 

<a href="../bar.html">Startseite</a>

Wichtige Hinweise für Verlinkungen

  • Immer die Dateiendung .html angeben.
  • Pfad korrekt angeben, sonst wird die Datei nicht gefunden.
  • Bei internen Links auf korrekte Ordnerstruktur achten.

Zusammenfassung

Der <a>-Tag ist eines der wichtigsten HTML-Elemente, da er die Navigation innerhalb einer Website ermöglicht. Durch korrekt verwendete Pfade und eine saubere Ordnerstruktur lässt sich eine intuitive Navigation umsetzen.
