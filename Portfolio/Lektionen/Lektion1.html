Lektion 1 – Einführung in JavaScript

Einführung

Willkommen zur ersten Vorlesung! In dieser Lektion geht es um die Grundlagen von Web und Server. Du lernst, wie das Client-Server-Modell funktioniert, wie Datenbanken eingebunden werden, wie HTML-Dokumente erzeugt werden und welche Rollen JavaScript und Skriptsprachen in modernen Webanwendungen spielen.

Einführungsbeispiel: Client & Server

Abbildung: Vereinfachte Darstellung einer Internetverbindung zwischen Client und Server

Das Internet basiert auf einer Kommunikation zwischen Client und Server. Der Client sendet eine Anfrage an den Server – dieser verarbeitet sie, ruft ggf. Daten aus einer Datenbank ab, generiert ein passendes HTML-Dokument und schickt dieses an den Browser des Nutzers zurück.

  • Es wäre unmöglich, für jeden möglichen Suchbegriff eine fertige HTML-Datei zu speichern.
  • Stattdessen liegen alle Informationen in einer Datenbank.
  • Die Datenbank liefert eine Liste von Treffern zurück.
  • Der Server erzeugt daraus ein HTML-Dokument und sendet es an den Client.

Serverseitige Verarbeitung

Abbildung: Serverseitige Verarbeitung mit Anbindung einer Datenbank

Serverseitige Verarbeitung bedeutet, dass die Logik auf dem Server ausgeführt wird – oft mit direkter Anbindung an eine Datenbank. Häufig eingesetzte Programmiersprachen für diesen Zweck sind:

  • PHP
  • Perl
  • Java (z. B. Servlets)

Diese Programmiersprachen ermöglichen es, Anfragen zu verarbeiten, Daten zu manipulieren und dynamische Webseiten zu generieren.

Clientseitige Verarbeitung

Abbildung: Clientseitige Verarbeitung

Die Client-seitige Verarbeitung findet im Browser des Nutzers statt. Hier wird JavaScript verwendet, um die Oberfläche interaktiv zu gestalten, Eingaben zu prüfen oder Inhalte dynamisch zu verändern.

  • Echtzeit-Validierung von Formularen
  • Animationen oder Pop-up-Fenster
  • Interaktive Navigationselemente

Software-Architektur von Webanwendungen

Client-seitige Programmierung („Rich Client“)

Abbildung: Client-seitig – Rich-thick Client

Die gesamte Logik läuft im Browser.
Vorteile: Technische Umgebung des Benutzers kann berücksichtigt werden.
Nachteile: Der Nutzer kann Code deaktivieren.

Server-seitige Programmierung

Abbildung: Software-Architektur von Webanwendungen – gesamte Programmierung server-seitig

Anzeige und Dokument im Webclient → Interaktion mit dem Webserver. Der Webserver interagiert mit Java Servlets, CGI-Anwendungen und PHP-Anwendungen.
Vorteile: Man ist nicht darauf angewiesen, dass der Nutzer die clientseitige Verarbeitung einschaltet.
Nachteile: Technische Umgebung des Benutzers wird nicht berücksichtigt.

Erweiterung um Programme oder Skripte

Abbildung: Software-Architektur – client-seitige und server-seitige Erweiterung um Programme oder Skripte

Beschreibung: Anzeige und Dokument im Webclient → Interaktion mit dem Webserver. Der Webserver interagiert mit Java Servlets, CGI-Anwendungen und PHP-Anwendungen.

  • Vorteile: Vorteile von Client- und Serverseitiger Verarbeitung können genutzt werden.

Programmiersprachen und Skriptsprachen

  • Programmiersprachen: Java, C++, C, Assembler, Fortran
  • Auszeichnungssprachen: HTML, XML
  • Skriptsprachen: PHP, Perl, JavaScript, Python, Ruby

Übersetzung von Programmiersprachen

  • Ein Übersetzer (Compiler) wandelt Quellcode in Maschinensprache um.
  • Zielsprache ist meist eine Prozessor- oder virtuelle Maschinensprache (z. B. JVM).
  • Übersetzte Sprachen besitzen eine starke statische Semantik.

Interpretation von Sprachen

  • Ein Interpreter liest und führt ein Programm direkt aus.
  • Einfache Struktur, keine statische Semantik.
  • Typregeln werden erst bei der Ausführung geprüft.
  • Interpreter können plattformunabhängig bereitgestellt werden.
  • Interpretation ist oft langsamer als die Ausführung übersetzter Programme.

Skriptsprachen

  • Einfache Aufgaben lösen.
  • Bestehende Funktionen kombinieren.
  • Leicht erlernbar und dynamisch typisiert.
  • Direkt interpretiert – keine Übersetzung nötig.

JavaScript

JavaScript ist eine leichtgewichtige Skriptsprache, die speziell für HTML entwickelt wurde. Sie wird direkt im Browser interpretiert und ermöglicht interaktive Webseiten (DHTML, Web 2.0).

Einbindung von JavaScript

<script type="text/javascript">
  // JavaScript Code
</script>

Beispiel: Eine Meldung ausgeben

<script type="text/javascript">
  alert("Das ist JavaScript!");
</script>

Klicke auf den Knopf, um das JavaScript-Beispiel in Aktion zu sehen.

Beispiel: Rechnen in JavaScript

<script type="text/javascript">
  alert(17 + 5 * 3);
</script>

Interaktivität durch Buttons

<form>
  <input type="button" value="Test" onclick="alert('Hallo!')">
</form>

Kommentare

// Einzeiliger Kommentar
/* Mehrzeiliger Kommentar */

noscript-Tag

Falls JavaScript deaktiviert ist, kann der <noscript>-Tag Benutzer informieren oder Alternativen anzeigen.

Merke

JavaScript macht Webseiten lebendig – es verbindet die statische HTML-Struktur mit dynamischem Verhalten.
